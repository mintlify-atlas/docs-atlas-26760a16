title: Services
description: Manage services, group bookings, and service categories

Create and manage individual services and group booking services with pricing, duration, and inventory tracking.

## Create Service

<Note>
Requires authentication.
</Note>

Create a new individual booking service with multiple phases.

```bash POST /api/v1/merchant/services
curl -X POST 'https://api.example.com/api/v1/merchant/services' \
  --cookie "session_token=your_session_token" \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Haircut and Style",
    "description": "Professional haircut with styling",
    "color": "#FF5733",
    "price": {"amount": 5000, "currency": "USD"},
    "cost": {"amount": 1000, "currency": "USD"},
    "price_type": "fixed",
    "category_id": 1,
    "is_active": true,
    "settings": {
      "cancel_deadline": 1440,
      "booking_window_min": 60,
      "booking_window_max": 10080,
      "buffer_time": 15
    },
    "phases": [
      {
        "name": "Haircut",
        "sequence": 1,
        "duration": 30,
        "phase_type": "active"
      },
      {
        "name": "Styling",
        "sequence": 2,
        "duration": 15,
        "phase_type": "active"
      }
    ],
    "used_products": [
      {"id": 1, "amount_used": 50},
      {"id": 2, "amount_used": 20}
    ]
  }'
```

<ParamField body="name" type="string" required>
  Service name
</ParamField>

<ParamField body="description" type="string" optional>
  Detailed service description
</ParamField>

<ParamField body="color" type="string" required>
  Hex color code for calendar display (must be valid hex color)
</ParamField>

<ParamField body="price" type="object" optional>
  Service price
  <Expandable title="Properties">
    <ParamField body="amount" type="integer">
      Price in smallest currency unit (cents)
    </ParamField>
    <ParamField body="currency" type="string">
      Three-letter currency code
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="cost" type="object" optional>
  Service cost for internal tracking
</ParamField>

<ParamField body="price_type" type="string" required>
  Pricing type: "fixed", "free", "variable", or "starting_at"
</ParamField>

<ParamField body="category_id" type="integer" optional>
  Service category ID for organization
</ParamField>

<ParamField body="is_active" type="boolean" required>
  Whether service is available for booking
</ParamField>

<ParamField body="settings" type="object" required>
  Service-specific scheduling settings (overrides merchant defaults if set)
  <Expandable title="Properties">
    <ParamField body="cancel_deadline" type="integer" optional>
      Cancellation deadline in minutes
    </ParamField>
    <ParamField body="booking_window_min" type="integer" optional>
      Minimum advance booking time in minutes
    </ParamField>
    <ParamField body="booking_window_max" type="integer" optional>
      Maximum advance booking time in minutes
    </ParamField>
    <ParamField body="buffer_time" type="integer" optional>
      Buffer time after service in minutes
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="phases" type="array" required>
  Service phases defining duration and type
  <Expandable title="Phase object">
    <ParamField body="name" type="string" required>
      Phase name
    </ParamField>
    <ParamField body="sequence" type="integer" required>
      Order sequence (1, 2, 3...)
    </ParamField>
    <ParamField body="duration" type="integer" required>
      Duration in minutes (1-1440)
    </ParamField>
    <ParamField body="phase_type" type="string" required>
      "active" for work time or "wait" for waiting time
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="used_products" type="array" required>
  Products consumed by this service
  <Expandable title="Product usage object">
    <ParamField body="id" type="integer" required>
      Product ID
    </ParamField>
    <ParamField body="amount_used" type="integer" required>
      Quantity used per service (0-1000000)
    </ParamField>
  </Expandable>
</ParamField>

### Response

Returns `201 Created` on success.

## Update Service

<Note>
Requires authentication.
</Note>

Update an existing service.

```bash PUT /api/v1/merchant/services/{id}
curl -X PUT 'https://api.example.com/api/v1/merchant/services/1' \
  --cookie "session_token=your_session_token" \
  --header 'Content-Type: application/json' \
  --data '{
    "id": 1,
    "name": "Premium Haircut",
    "description": "Premium haircut with styling",
    "color": "#FF5733",
    "price": {"amount": 7500, "currency": "USD"},
    "cost": {"amount": 1500, "currency": "USD"},
    "price_type": "fixed",
    "category_id": 1,
    "is_active": true,
    "settings": {},
    "phases": [
      {
        "id": 1,
        "service_id": 1,
        "name": "Haircut",
        "sequence": 1,
        "duration": 45,
        "phase_type": "active"
      }
    ]
  }'
```

<ParamField path="id" type="integer" required>
  Service ID
</ParamField>

Body parameters are the same as create, with phases including their IDs.

## Group Booking Services

Group services allow multiple participants to book the same time slot.

### Create Group Service

```bash POST /api/v1/merchant/services/group
curl -X POST 'https://api.example.com/api/v1/merchant/services/group' \
  --cookie "session_token=your_session_token" \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Yoga Class",
    "description": "Group yoga session",
    "color": "#4CAF50",
    "price": {"amount": 2500, "currency": "USD"},
    "cost": {"amount": 500, "currency": "USD"},
    "price_type": "fixed",
    "duration": 60,
    "category_id": 2,
    "min_participants": 3,
    "max_participants": 15,
    "is_active": true,
    "settings": {},
    "used_products": []
  }'
```

<ParamField body="min_participants" type="integer" optional>
  Minimum participants required
</ParamField>

<ParamField body="max_participants" type="integer" required>
  Maximum participants allowed
</ParamField>

<ParamField body="duration" type="integer" required>
  Session duration in minutes
</ParamField>

### Update Group Service

```bash PUT /api/v1/merchant/services/group/{id}
```

### Get Group Service

```bash GET /api/v1/merchant/services/group/{id}
curl -X GET 'https://api.example.com/api/v1/merchant/services/group/1' \
  --cookie "session_token=your_session_token"
```
