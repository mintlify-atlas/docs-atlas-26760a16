Manage your merchant configuration including business information, scheduling settings, business hours, and calendar preferences.

## Get Settings

<Note>
Requires authentication. Accessible by Staff, Admin, and Owner roles.
</Note>

Retrieve all merchant settings including business hours and location information.

```bash GET /api/v1/merchant/settings
curl -X GET 'https://api.example.com/api/v1/merchant/settings' \
  --cookie "session_token=your_session_token"
```

<ResponseField name="merchant_name" type="string">
  Business name
</ResponseField>

<ResponseField name="contact_email" type="string">
  Contact email address
</ResponseField>

<ResponseField name="introduction" type="string">
  Brief introduction or tagline
</ResponseField>

<ResponseField name="announcement" type="string">
  Current announcement or notice
</ResponseField>

<ResponseField name="about_us" type="string">
  Detailed business description
</ResponseField>

<ResponseField name="parking_info" type="string">
  Parking instructions for customers
</ResponseField>

<ResponseField name="payment_info" type="string">
  Payment information and accepted methods
</ResponseField>

<ResponseField name="cancel_deadline" type="integer">
  Cancellation deadline in minutes before appointment
</ResponseField>

<ResponseField name="booking_window_min" type="integer">
  Minimum advance booking time in minutes
</ResponseField>

<ResponseField name="booking_window_max" type="integer">
  Maximum advance booking time in minutes
</ResponseField>

<ResponseField name="buffer_time" type="integer">
  Buffer time between appointments in minutes
</ResponseField>

<ResponseField name="timezone" type="string">
  IANA timezone identifier (e.g., "America/New_York")
</ResponseField>

<ResponseField name="business_hours" type="object">
  Map of day (0-6, Sunday=0) to array of time slots

    <ResponseField name="start_time" type="string">
      Start time in HH:MM:SS format
    </ResponseField>
    <ResponseField name="end_time" type="string">
      End time in HH:MM:SS format
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="location_id" type="integer">
  Primary location ID
</ResponseField>

<ResponseField name="country" type="string" optional>
  Country name
</ResponseField>

<ResponseField name="city" type="string" optional>
  City name
</ResponseField>

<ResponseField name="postal_code" type="string" optional>
  Postal or ZIP code
</ResponseField>

<ResponseField name="address" type="string" optional>
  Street address
</ResponseField>

<ResponseField name="formatted_location" type="string">
  Complete formatted address
</ResponseField>

## Update Settings

<Note>
Requires authentication. Accessible by Staff, Admin, and Owner roles.
</Note>

Update merchant settings and business hours.

```bash PATCH /api/v1/merchant/settings
curl -X PATCH 'https://api.example.com/api/v1/merchant/settings' \
  --cookie "session_token=your_session_token" \
  --header 'Content-Type: application/json' \
  --data '{
    "introduction": "Welcome to our salon",
    "announcement": "Spring special: 20% off all services",
    "about_us": "Professional hair and beauty services since 2020",
    "parking_info": "Free parking in rear lot",
    "payment_info": "We accept cash, credit cards, and mobile payments",
    "cancel_deadline": 1440,
    "booking_window_min": 60,
    "booking_window_max": 10080,
    "buffer_time": 15,
    "business_hours": {
      "1": [{"start_time": "09:00:00", "end_time": "17:00:00"}],
      "2": [{"start_time": "09:00:00", "end_time": "17:00:00"}],
      "3": [{"start_time": "09:00:00", "end_time": "17:00:00"}],
      "4": [{"start_time": "09:00:00", "end_time": "17:00:00"}],
      "5": [{"start_time": "09:00:00", "end_time": "20:00:00"}]
    }
  }'
```

<ParamField body="introduction" type="string" required>
  Brief introduction or tagline
</ParamField>

<ParamField body="announcement" type="string" required>
  Current announcement (can be empty string)
</ParamField>

<ParamField body="about_us" type="string" required>
  Detailed business description
</ParamField>

<ParamField body="parking_info" type="string" required>
  Parking instructions
</ParamField>

<ParamField body="payment_info" type="string" required>
  Payment information
</ParamField>

<ParamField body="cancel_deadline" type="integer" required>
  Cancellation deadline in minutes
</ParamField>

<ParamField body="booking_window_min" type="integer" required>
  Minimum advance booking time in minutes
</ParamField>

<ParamField body="booking_window_max" type="integer" required>
  Maximum advance booking time in minutes
</ParamField>

<ParamField body="buffer_time" type="integer" required>
  Buffer time between appointments in minutes
</ParamField>

<ParamField body="business_hours" type="object" required>
  Map of day (0-6) to array of time slot objects with start_time and end_time
</ParamField>

## Get Normalized Business Hours

<Note>
Requires authentication. Accessible by Staff, Admin, and Owner roles.
</Note>

Get simplified business hours showing only the earliest start and latest end time for each day.

```bash GET /api/v1/merchant/settings/business-hours/normalized
curl -X GET 'https://api.example.com/api/v1/merchant/settings/business-hours/normalized' \
  --cookie "session_token=your_session_token"
```

Returns a map of day number to single time slot:

```json
{
  "1": {"start_time": "09:00:00", "end_time": "17:00:00"},
  "2": {"start_time": "09:00:00", "end_time": "17:00:00"}
}
```

## Check URL Availability

<Note>
Requires authentication. Accessible by Staff, Admin, and Owner roles.
</Note>

Check if a merchant URL name is available.

```bash POST /api/v1/merchant/check-url
curl -X POST 'https://api.example.com/api/v1/merchant/check-url' \
  --cookie "session_token=your_session_token" \
  --header 'Content-Type: application/json' \
  --data '{"merchant_name": "my-salon"}'
```

<ParamField body="merchant_name" type="string" required>
  Desired URL-friendly merchant name
</ParamField>

Returns `200 OK` if available, or `409 Conflict` if already taken:

```json
{
  "error": {
    "message": "Merchant URL is not unique",
    "merchant_url": "suggested-alternative"
  }
}
```

## Delete Merchant

<Warning>
Requires Owner role only. This action is irreversible.
</Warning>

Permanently delete the merchant account and all associated data.

```bash DELETE /api/v1/merchant
curl -X DELETE 'https://api.example.com/api/v1/merchant' \
  --cookie "session_token=your_session_token"
```

Returns `200 OK` on successful deletion.
